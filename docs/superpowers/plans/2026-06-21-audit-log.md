# Audit Log — Implementation Plan

> Executed directly (not subagent-driven). Spec: `docs/superpowers/specs/2026-06-21-audit-log-design.md`.

**Goal:** Tamper-evident, append-only audit log of sensitive mutations, with a superadmin view + chain verification.

**Tech:** Cloudflare Workers (Hono) + D1; React admin dashboard.

Shipped in two PRs: **A = backend** (schema, service, wiring, endpoints), **B = dashboard tab**.

---

## PR A — Backend

### Task A1: Schema + migration + plumbing
- Create `packages/api/src/db/migration-audit-log.sql` — `audit_log` table + 4 indexes (per spec).
- Add the same `CREATE TABLE`/indexes to `packages/api/src/db/schema.sql`.
- Register `migration-audit-log.sql` in `db/migrations-index.ts` (`import` + `MIGRATIONS` entry).
- Add `'audit_log'` to `services/backup.ts` `BACKUP_TABLES`.

### Task A2: `services/audit.ts`
- `REDACT = /pin|password|secret|token|hash|api[_-]?key/i`; `redactValue(field, v)`.
- `diffRecords(before, after, fields?)` → `{ field: {from,to} }` for changed fields only, redacted.
- `auditActorFromContext(c)` → `{ actor: {userId,role,label}, ip }` from `c.get('session')` (try/catch) + `cf-connecting-ip`.
- `recordAudit(env, ctx, input)` — non-fatal, awaited:
  1. read last row `SELECT seq, hash FROM audit_log ORDER BY seq DESC LIMIT 1`;
  2. `seq = last+1`, `at = new Date().toISOString()`, `prev_hash = last.hash ?? ''`;
  3. `canonical = JSON.stringify({seq,at,actor_user_id,actor_role,action,entity_type,entity_id,summary,changes})`;
  4. `hash = await sha256Hex(canonical + prev_hash)` (reuse from migrations-index);
  5. INSERT; on `UNIQUE`/seq conflict retry ≤5×; on final failure `console.error` + `alertAdminError`, swallow.
- `verifyChain(env)` → walk by seq asc, recompute hash, compare; return `{ok, checked, brokenAtSeq?}`.
- `listAudit(env, filters)` → cursor (seq desc) paginated query with optional `entity_type/action/actor_user_id/from/to/q`.

### Task A3: Wire mutation handlers (await `recordAudit` after the write; capture before/after for updates)
- `routes/users.ts`: create → `user.create`; update → `user.update` (and `user.role_change` when role differs) using before/after `SELECT *`; delete → `user.deactivate`.
- `routes/admin-nss.ts`: create → `nss.create`.
- `routes/admin-interns.ts`: create → `intern.create`.
- `routes/admin-directorates.ts`: directorate create/update, officer create/update, receiver add/remove, `set_primary` (PUT reception_officer_id), telegram link-token/revoke.
- `routes/admin-holidays.ts`: create/delete.
- `routes/admin-settings.ts`: update (diff old/new settings; override PIN redacted).
- `routes/kiosk.ts`: `override.use` on both gates (ID + office-closed) — actor null/'reception (shared PIN)', summary names gate + visit.
- `routes/bulk-import.ts`: `/users` → one `users.bulk_import` summary (imported/skipped counts).
- `routes/admin-migrations.ts`: `/run` → `migrations.run` (applied/skipped).

### Task A4: Endpoints
- `routes/admin-audit.ts`: `GET /` (superadmin; filters + cursor) and `GET /verify` (superadmin).
- Mount in `index.ts`: `app.route('/api/admin/audit', adminAuditRoutes)`.

### Task A5: Gates + ship
- `tsc --noEmit` (api), commit, PR, merge, deploy. Apply migration via the **Run pending migrations** button. Verify via curl/endpoint with a session.

## PR B — Dashboard

### Task B1: `components/admin/AuditLogTab.tsx`
- Query `GET /admin/audit` (filters: entity type, action, date range, text; cursor "load more").
- Table: time · actor (name + role) · action badge · entity · summary; row expands to show `changes` diff (from→to, redacted values shown as `[redacted]`).
- **Verify integrity** button → `GET /admin/audit/verify` → ✓ intact (N checked) / ✗ broken at seq N.

### Task B2: Wire into `AdminPage.tsx`
- Add an **Audit Log** tab (superadmin only) rendering `<AuditLogTab />`.

### Task B3: Gates + ship
- `tsc` + web build, commit, PR, merge, deploy. Verify in dashboard: actions appear, diffs redacted, verify button works; edit a row in D1 → verify reports broken.

## No placeholders / consistency
- Action strings are fixed (see spec catalog). `recordAudit` signature is identical across all call sites. `changes` is `null` for create/delete, a diff object for updates.
