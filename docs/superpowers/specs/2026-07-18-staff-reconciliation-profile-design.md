# Staff Reconciliation, Phone & Profile Design
**Date:** 2026-07-18  
**Status:** Approved

## Problem

1. `officers.staff_id` is NULL for all officers imported via `migration-staff-officers.sql` — the column was added later and never backfilled. This means "Provision Missing Accounts" finds nothing to match.
2. `users` has no `phone` column — contact numbers from the Excel are not stored on Staff Attendance accounts.
3. The VMS admin has no visibility into which officers have Staff Attendance accounts, real emails, or phone numbers.
4. Staff have no self-service path to update their own phone or email after first login.

**Constraint:** The Staff ID + PIN login flow (`POST /auth/pin-login`) is not touched.

---

## Source of Truth

Excel workbook: `docs/Staff List/UPDATED STAFFLIST 2026 FOR 39.xlsx`  
Primary sheet: `Updated stafflist OHCS`  
- Column B = Staff No. (numeric)  
- Column C/D/E = First / Surname / Middle name  
- Column R = Phone No.  
- 166 staff rows; 165 have phone numbers

---

## Section 1 — Data Model

### `migration-users-phone.sql`
```sql
ALTER TABLE users ADD COLUMN phone TEXT;
```

### `migration-officers-staff-id-backfill.sql` (generated)
Auto-generated by `scripts/gen-staff-reconciliation-migration.py` from the Excel.

For each numeric staff number in column B:
```sql
UPDATE officers
SET staff_id = '<staff_no>', phone = '<phone>'
WHERE id = 'off_<staff_no>'
  AND (staff_id IS NULL OR phone IS NULL OR phone != '<phone>');
```

For any Excel row whose `off_XXXXXX` ID does not exist in the DB:
```sql
INSERT OR IGNORE INTO officers (id, name, title, directorate_id, phone, staff_id, is_available)
VALUES ('off_<staff_no>', '<name>', '<grade>', '<dir_id>', '<phone>', '<staff_no>', 1);
```

**Skips:** non-numeric column B values (section headers), the `off_GRA` edge-case officer.

Both migrations registered in `migrations-index.ts` and applied via Settings → Run Migrations.

`AUDITED_USER_FIELDS` in `users.ts` gains `'phone'`.

## Section 2 — API Changes

### 2a. Provision upgrade (`POST /users/provision-from-officers`)
When creating a `users` row from an officer, copy `officers.phone → users.phone`.

### 2b. Staff self-update (`PATCH /auth/profile`)
Authenticated endpoint — any role.

**Accepted fields:**
| Field | Auth required | Notes |
|-------|--------------|-------|
| `phone` | None | Saved immediately |
| `email` | `current_pin` (4–6 digits) | PIN verified via existing `verifyPin`; on success bumps `session_epoch` (other devices log out; current session re-issued cleanly) |

Returns updated user object. Audited as `user.profile_update`.

PIN lockout logic mirrors `/auth/pin-login` (same `getPinLock` / `recordPinFailure` / `clearPinLock` calls).

**Not editable here:** name, staff_id, role (admin-managed only).

### 2c. Admin officer edit propagates phone (`PUT /admin/directorates/officers/:id` in `admin-directorates.ts`)
After updating `officers.phone`, find the linked `users` row by `staff_id` and update `users.phone` there too — keeping option C in sync.

Phone sync is **one-directional at the admin level**: `officers.phone → users.phone` only when the admin edits the officer record. If an admin edits `users.phone` directly in the Users tab, it does NOT write back to `officers`. This is acceptable — `officers.phone` is for VMS reception routing; `users.phone` is for HR/attendance contact.

### 2d. Users list includes phone (`GET /users`)
Add `u.phone` to the SELECT.

## Section 3 — Excel Reconciliation Script

**File:** `scripts/gen-staff-reconciliation-migration.py`

**Reads:** `docs/Staff List/UPDATED STAFFLIST 2026 FOR 39.xlsx` (sheet: "Updated stafflist OHCS")

**Writes:** `packages/api/src/db/migration-officers-staff-id-backfill.sql`

**Logic:**
1. Parse all rows where column B is a positive integer.
2. Build name as `<First> <Middle_initial> <Surname>` (trimmed, collapsed whitespace).
3. Map grade → directorate using the same abbreviation lookup as `migration-staff-officers.sql`.
4. Emit `UPDATE` for existing officers; emit `INSERT OR IGNORE` for any whose `off_<staff_no>` ID is not in the known officer ID list.
5. Header comment: `-- Auto-generated by scripts/gen-staff-reconciliation-migration.py — do not edit by hand`

**Known edge cases:**
- `off_GRA` (Fred Charles Anson) — no numeric staff number; script skips.
- Rows with merged/section-header cells — column B is non-integer; script skips.
- Duplicate staff numbers across sheets — script deduplicates, primary sheet wins.

## Section 4 — Admin UI Changes

### 4a. Users tab — phone column + readiness badge

**Readiness badge** (replaces plain "Active/Inactive" in Status column):

| Badge | Criteria |
|-------|----------|
| 🟢 Ready | `is_active=1` + email not `@ohcs.internal` + `phone` set |
| 🟡 Partial | `is_active=1` but missing phone or has placeholder email |
| 🔴 Inactive | `is_active=0` |

**Header summary line** (below the card title):  
`N ready · M partial · P no account`  
where "no account" = officers with `staff_id` but no matching `users` row (fetched via a separate lightweight query).

**"Provision Missing Accounts" button** promoted from Bulk Import tab into the Users tab header area. Calls the existing `/users/provision-from-officers` endpoint.

**Phone column** added between Grade and Role in the users table.

### 4b. Org Entities tab — officers list

Two new columns added to the officers table:

| Column | Content |
|--------|---------|
| Staff ID | `officers.staff_id` or `—` |
| SA Account | Green "Linked" chip if `users.staff_id` match exists; grey "None" if not |

**Officer edit modal** gains a **Phone** field (the column already exists in the schema; this just exposes it for editing — currently missing from the form).

### 4c. Scope

No new admin tabs. All changes land in existing **Users** and **Org Entities** tabs.

## Section 5 — Staff Profile Page

**Route:** `/profile`  
**Access:** All authenticated roles  
**Nav entry:** Added to `MORE_ITEMS` in `BottomNav.tsx` (the 4 main slots are full) and to `Sidebar.tsx` for desktop. Icon: `UserCircle` from lucide-react.

### First-login nudge

The existing `pin_acknowledged === false` banner gains a second CTA alongside "Change PIN": **"Update contact details"** → links to `/profile`.

### Layout

**Card 1 — Identity (read-only)**
- Name, Staff ID, Role, Directorate

**Card 2 — Contact (editable)**
- Phone field — saves on submit, no PIN required
- Email field — shows current value; when user edits it, a PIN confirmation input slides in below; submit sends `{ email, current_pin }` to `PATCH /auth/profile`

**Feedback:**
- Success toast: "Profile updated"
- On email change: "Email updated. Other devices have been signed out."
- On PIN failure: same error copy as login ("Invalid PIN" with remaining attempts from lockout state)

### What is NOT editable here
Name, Staff ID, role, directorate — these remain admin-managed only and are shown as read-only chips.

## Sequence: recommended go-live order

1. Run `scripts/gen-staff-reconciliation-migration.py` → review generated SQL
2. Apply both migrations (phone column + staff_id backfill) via Settings → Run Migrations
3. Click "Provision Missing Accounts" in Users tab — creates Staff Attendance accounts for all officers with staff IDs
4. Staff log in with Staff ID + PIN, land on Profile page, fill in phone/real email
5. Admin can see readiness badges drain from amber to green over time
